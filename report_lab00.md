## Условие

Необходимо сформировать два набора данных для приложений машинного
обучения. Первый датасет должен представлять из себя табличный набор данных для
задачи классификации. Второй датасет должен быть отличен от первого, и может
представлять из себя набор изображений, корпус документов, другой табличный
датасет или датасет из соревнования Kaggle, предназначенный для решения
интересующей вас задачи машинного обучения. Необходимо провести анализ обоих
наборов данных, поставить решаемую вами задачу, определить признаки необходимые
для решения задачи, в случае необходимости заняться генерацией новых признаков,
устранением проблем в данных, визуализировать распределение и зависимость
целевого признака от выбранных признаков. В отчете описать все проблемы, с
которыми вы столкнулись и выбранные подходы к их решению.

## Подготовка данных

Датасеты взяты из kaggle.

Первый датасет: https://www.kaggle.com/ang3loliveira/malware-analysis-datasets-top1000-pe-imports

В нем требуется определить вредоносное ПО по используемым функциям WinAPI. 

Для удобства хеш-суммы были записаны в десятичной системе счисления.

Второй датасет: https://www.kaggle.com/aiaiaidavid/cardio-data-dv13032020

В нем требуется определить наличие сердечного заболевания по описанию человека.

Перед подсчетом оценки работы моделей машинного обучения данные стандартизируются.

## Анализ датасета.

Просматривается общая информация о датасете, его описательная статистика и количество уникальных элементов в каждом столбце. 
Для отображения графиков используется библиотека plotly. Визуализируется распределение данных.

## Алгоритмы машинного обучения

Логистическая регрессия, KNN, SVM, дерево решений и случайный лес взяты из sklearn. 

Для оценки обучения используется коэффициент корреляции Мэтьюса, потому что он объективен для сравнения разных моделей на разных задачах.

## Сравнение алгоритмов машинного обучения

Лучшим по точности оказался SVM. 

Логистическая регрессия и случайный лес оказались неплохими для обучения. 

KNN и дерево решений имеют неоднозначный результат на рассматриваемых датасетах. Определенно, они не лучшие.

## Выводы

Так как входные данные могут быть очень большими, то лучше их обрезать, чтобы обработка была быстрее. 
Хеш-сумму для удобства лучше перевести в десятичное число. 
Предупреждения в python можно выключить. 
Коэфициент корреляции Мэьтюса принимает значения от -1 до 1 (от худшего к лучшему). 
Он удобен для использования в разных задачах классификации. 
Если данные в датасете разделены точкой с запятой, то он некорректно считывается. 
Поэтому необходимо заменить ее на запятую.
